## About the Solution

Cattle Vaccination is a worked reference architecture that teams can adapt to their own service domain. It demonstrates a common **Applicant → Submission → Regulator** flow where users submit information, an internal team assesses and decides, and notifications keep participants informed as work moves through its lifecycle.

The model is organised into three delivery contexts: **Submission** (the external journey and API), **Regulation** (internal case handling and decisioning), and **Notifications** (event-driven communications). A planned **Post-decision self-service** context is included to show how the architecture can evolve beyond an initial release.

## One Model, Every View

These docs are generated from a single [LikeC4](https://likec4.dev/) model that lives next to the code. Every container, component, deployment node and journey on this site is rendered from that one source — so the diagrams stay in step with each other, and with reality, by construction.

- **Click into anything.** Containers open into components; components reveal the relationships they take part in.
- **Filter by time.** Each structural view has Current, Future and Evolution tabs driven by `#legacy` and `#future` tags on individual elements.
- **Trace a flow.** Journey diagrams are dynamic views over the same static model, so a renamed component updates everywhere it appears.

## Service Snapshot

<CardGrid>
  <Card title="Bounded contexts" icon="puzzle">
    **License Submission** — applicant-facing frontend and API for submitting licence applications.

    **License Regulation** — internal case-management platform (e.g. Dynamics 365) where regulators assess and decide on applications.

    **License Notifications** — event-driven integrations that dispatch emails and other notifications when application status changes.
  </Card>
  <Card title="Planned" icon="seti:plan">
    **Permit Holder** actor and self-service portal — closing the loop so licence holders can manage renewals, variations and ongoing obligations.
  </Card>
  <Card title="Runs on" icon="seti:config">
    Defra **Core Delivery Platform** (CDP) on AWS — public VPC for the applicant UI, protected VPC for backend APIs and the regulation platform, with an ALB and API Gateway at the boundary.
  </Card>
  <Card title="Integrates with" icon="seti:cog">
    **Defra ID** for OIDC authentication, **GOV.UK Notify** for email and SMS notifications, and a legacy **Azure-hosted third-party API** being phased out as the service matures.
  </Card>
</CardGrid>
