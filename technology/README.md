# Technology View

The Technology area is the entry point for the architecture of the Cattle Vaccination service. It is organised as three lenses on the same service — each holding the diagrams, narrative and decisions for that lens — plus a quality assurance area for stage-gate artefacts.

## Where This is Published

The Technology area is published to two surfaces in parallel so anyone can read it regardless of whether they have GitHub access. The content is identical between the two; the surfaces differ only in diagram fidelity and navigation:

- **GitHub Pages** (link to be added) - for engineers and others with GitHub access. Interactive C4 diagrams with clickable navigation, zoom and pan.
- **Confluence** (link to be added) - for the larger group of readers who don't typically have GitHub access: business analysts, product owners, user researchers, assurance, design authority, programme leadership, partner organisations. Static, lower-fidelity renders of the same diagrams; sits inside the team's existing service hierarchy.

## What's in Here

- [Current State Views](./current-state-views/README.md) — how the system is structured, how it evolves, where it runs, and how end-to-end journeys move through it.
- [Data Architecture](./data-architecture/README.md) — what data the service owns, where it lives, how it flows through the lifecycle, and how the data landscape changes over time.
- [Integration Architecture](./integration-architecture/README.md) — interfaces with other systems, the patterns used to integrate with them, and the contracts that govern those integrations.
- [Quality Assurance](./quality-assurance-view/README.md) — the Solution Overview living document and stage-gate readiness checklists from discovery through live.

Each area has its own README with the detailed treatment; this page is the map.
